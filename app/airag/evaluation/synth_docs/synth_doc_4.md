# Project Delivery Reference B

## 1. velocity

Team velocity is the amount of work completed in an iteration and is useful for forecasting that same team's capacity.

## 2. backlog refinement

Backlog refinement clarifies, splits, estimates, and reorders upcoming work before implementation.

## 3. acceptance criteria

Acceptance criteria are testable conditions that a work item must satisfy to be accepted as complete.

## 4. burn-down chart

A burn-down chart displays remaining work over time toward the end of an iteration or release.

## 5. resource leveling

Resource leveling adjusts task dates to resolve over-allocation of limited people or equipment.

## 6. lead time

Lead time measures elapsed time from a request being made until the requested value is delivered.

## 7. cycle time

Cycle time measures elapsed time from starting work on an item until that item is finished.

## 8. definition of done

A definition of done is a shared quality checklist applied to every completed increment.

## 9. RACI

A RACI matrix identifies who is responsible, accountable, consulted, and informed for each activity.

## 10. timebox

A timebox is a fixed maximum period allocated to an activity, after which the activity stops or is reviewed.
